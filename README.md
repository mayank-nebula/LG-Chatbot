•	It was an embedded media file within a web page (so that the web page load, and some activities on the page were trackable, even if the PDF wasn't). 
•	Using a WP plugin, the team could simply "replace" the media file when updates were made, so the embed link remained the same while the page would display the updated document.
•	It used the WordPress password protect feature to ask for a specific password, to prevent indexing and strangers from finding it.
•	For the "gated download" version, it required an email address to view the content as well.
•	That email address then triggered a workflow to the CRM, to either record a media kit view on an existing record or create a new record - both based on the email address.
